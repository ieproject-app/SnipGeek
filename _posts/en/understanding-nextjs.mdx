Next.js provides a robust foundation for building server-rendered React applications, static websites, and more. It comes with a rich set of features out-of-the-box that streamline development.

## Core Concepts

Understanding these core concepts is crucial for effectively using Next.js.

### Pages and Routing

In Next.js, a page is a React Component exported from a `.js`, `.jsx`, `.ts`, or `.tsx` file in the `app` directory. Each page is associated with a route based on its file name.

<Note>
The App Router, introduced in Next.js 13, is the recommended way to build new applications. It offers features like layouts, server components, and more.
</Note>

Here is an example of a simple page component in TypeScript:

```tsx
// src/app/about/page.tsx

export default function AboutPage() {
  return <h1>About Us</h1>;
}
```

This code creates a new page accessible at the `/about` URL. It's that simple!

### Data Fetching

Next.js offers different methods for data fetching, depending on your use case. You can fetch data on the server-side with Server Components, or on the client-side.

<Warning>
When fetching sensitive data, always do it on the server-side to avoid exposing API keys or other secrets to the browser.
</Warning>
