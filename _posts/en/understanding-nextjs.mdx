Next.js has revolutionized React development by providing a powerful framework with features like Server-Side Rendering (SSR), Static Site Generation (SSG), and now, the App Router with React Server Components (RSC).

## The App Router

The App Router, introduced in Next.js 13, is a paradigm shift. It allows you to build applications using a file-based routing system within the `app` directory. Each folder represents a route segment.

> **Note:** The App Router uses Server Components by default, which is a major performance benefit. This means less JavaScript is shipped to the client, resulting in faster initial page loads.

Here is a long code example to demonstrate how a code block should look and behave. This block should have a consistent background and be horizontally scrollable on smaller screens if the content overflows.

```tsx
// src/app/layout.tsx
import { Inter } from 'next/font/google';
import './globals.css';
import { ThemeProvider } from '@/components/theme-provider';
import { Analytics } from '@vercel/analytics/react';
 
const inter = Inter({ subsets: ['latin'] });
 
export const metadata = {
  title: 'My Awesome App',
  description: 'Generated by create next app, this is an example of a very long line of code that should cause the container to scroll horizontally instead of wrapping or overflowing badly.',
};
 
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className={inter.className}>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
          <Analytics />
        </ThemeProvider>
      </body>
    </html>
  );
}
```

## Client vs. Server Components

With the App Router, you can explicitly define components as either Client or Server Components.

- **Server Components** run exclusively on the server. They can directly access backend resources like databases and file systems.
- **Client Components** are interactive and run on the client. You opt into them by adding the `'use client'` directive at the top of your file.

> **Warning:** Be careful when fetching sensitive data. Server Components are a great place for this, but never expose secret keys or tokens to Client Components.
