Next.js is a React framework that gives you the best developer experience with all the features you need for production: hybrid static & server rendering, TypeScript support, smart bundling, route pre-fetching, and more. No config needed.

## The Building Blocks

At its core, Next.js provides a structured way to build React applications. It handles much of the tooling and configuration that can be a headache when starting a new project from scratch.

Here is a longer code example to demonstrate how `rehype-pretty-code` handles syntax highlighting and overflow. This should now be rendered with the `github-dark` theme and scroll horizontally on small screens.

```tsx
// src/app/layout.tsx
import { Inter } from 'next/font/google';
import './globals.css';
import { ThemeProvider } from '@/components/theme-provider';
import { Analytics } from '@vercel/analytics/react';
 
const inter = Inter({ subsets: ['latin'] });
 
export const metadata = {
  title: 'My Awesome App',
  description: 'Generated by create next app, and now it has a much longer description to see how the text wraps and how the entire code block behaves when its content is very wide.',
};
 
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className={inter.className}>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
          <Analytics />
        </ThemeProvider>
      </body>
    </html>
  );
}
```

An inline code snippet, like `const example = "hello"`, should have a simple grey background and not the full syntax highlighting.
