Next.js telah merevolusi pengembangan React dengan menyediakan kerangka kerja tangguh yang menangani semuanya, mulai dari server-side rendering hingga pembuatan situs statis. Dalam postingan ini, kita akan membedah arsitekturnya.

## App Router: Sebuah Paradigma Baru

Perubahan paling signifikan dalam versi terbaru Next.js adalah pengenalan App Router. Ini memanfaatkan React Server Components, memungkinkan pengembang membangun aplikasi kompleks dengan lebih sedikit JavaScript di sisi klien.

Berikut adalah contoh blok kode yang panjang untuk mendemonstrasikan pewarnaan sintaksis dan fungsionalitas gulir. Perhatikan latar belakang gelap dan scrollbar horizontal di layar yang lebih kecil.

```tsx
// src/app/layout.tsx
import { Inter } from 'next/font/google';
import './globals.css';
import { ThemeProvider } from '@/components/theme-provider';
import { Analytics } from '@vercel/analytics/react';

const inter = Inter({ subsets: ['latin'] });

export const metadata = {
  title: 'My Awesome App',
  description: 'Generated by create next app',
};

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className={inter.className}>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
          <Analytics />
        </ThemeProvider>
      </body>
    </html>
  );
}
```

<Note>
  Ini adalah komponen catatan! Ini bagus untuk menyorot informasi yang tidak kritis tetapi memberikan konteks yang membantu. Anda bisa menggunakan **Markdown** di dalamnya.
</Note>

## Server vs. Client Components

Memahami perbedaannya adalah kunci:
- **Server Components**: Berjalan di server, dapat mengakses database dan file, dan tidak interaktif.
- **Client Components**: Berjalan di klien, interaktif, dan menggunakan hooks seperti `useState` dan `useEffect`.

<Warning>
  Berhati-hatilah saat mengambil data di Client Components. Untuk menghindari terungkapnya kunci sensitif, Anda harus selalu memanggil rute API atau Server Action alih-alih mengambil data secara langsung.
</Warning>
