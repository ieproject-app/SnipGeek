Next.js menyediakan fondasi yang kokoh untuk membangun aplikasi React yang di-render di server, situs web statis, dan banyak lagi. Kerangka kerja ini hadir dengan serangkaian fitur bawaan yang memperlancar proses pengembangan.

## Konsep Inti

Memahami konsep-konsep inti ini sangat penting untuk menggunakan Next.js secara efektif.

### Halaman dan Perutean

Di Next.js, sebuah halaman adalah Komponen React yang diekspor dari file `.js`, `.jsx`, `.ts`, atau `.tsx` di dalam direktori `app`. Setiap halaman dikaitkan dengan sebuah rute berdasarkan nama filenya.

<Note>
App Router, yang diperkenalkan di Next.js 13, adalah cara yang direkomendasikan untuk membangun aplikasi baru. Fitur ini menawarkan hal-hal seperti layout, komponen server, dan lainnya.
</Note>

Berikut adalah contoh komponen halaman sederhana dalam TypeScript:

```tsx
// src/app/tentang/page.tsx

export default function HalamanTentang() {
  return <h1>Tentang Kami</h1>;
}
```

Kode ini membuat halaman baru yang dapat diakses di URL `/tentang`. Sangat sederhana!

### Pengambilan Data

Next.js menawarkan metode pengambilan data yang berbeda, tergantung pada kasus penggunaan Anda. Anda dapat mengambil data di sisi server dengan Komponen Server, atau di sisi klien.

<Warning>
Saat mengambil data sensitif, selalu lakukan di sisi server untuk menghindari terpaparnya kunci API atau rahasia lainnya ke browser.
</Warning>
