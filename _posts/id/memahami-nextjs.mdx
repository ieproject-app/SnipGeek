Next.js telah merevolusi pengembangan React dengan menyediakan kerangka kerja yang kuat dengan fitur-fitur seperti Server-Side Rendering (SSR), Static Site Generation (SSG), dan sekarang, App Router dengan React Server Components (RSC).

## App Router

App Router, yang diperkenalkan di Next.js 13, adalah sebuah perubahan paradigma. Ini memungkinkan Anda membangun aplikasi menggunakan sistem routing berbasis file di dalam direktori `app`. Setiap folder mewakili segmen rute.

> **Catatan:** App Router menggunakan Server Components secara default, yang merupakan keuntungan performa yang besar. Ini berarti lebih sedikit JavaScript yang dikirim ke klien, menghasilkan pemuatan halaman awal yang lebih cepat.

Berikut adalah contoh kode yang panjang untuk menunjukkan bagaimana seharusnya blok kode terlihat dan berfungsi. Blok ini seharusnya memiliki latar belakang yang konsisten dan dapat digulir secara horizontal di layar yang lebih kecil jika kontennya meluap.

```tsx
// src/app/layout.tsx
import { Inter } from 'next/font/google';
import './globals.css';
import { ThemeProvider } from '@/components/theme-provider';
import { Analytics } from '@vercel/analytics/react';
 
const inter = Inter({ subsets: ['latin'] });
 
export const metadata = {
  title: 'My Awesome App',
  description: 'Dibuat oleh create next app, ini adalah contoh baris kode yang sangat panjang yang seharusnya menyebabkan wadah dapat digulir secara horizontal daripada membungkus atau meluap dengan buruk.',
};
 
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className={inter.className}>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
          <Analytics />
        </ThemeProvider>
      </body>
    </html>
  );
}
```

## Client vs. Server Components

Dengan App Router, Anda dapat secara eksplisit mendefinisikan komponen sebagai Client atau Server Components.

- **Server Components** berjalan secara eksklusif di server. Mereka dapat langsung mengakses sumber daya backend seperti database dan sistem file.
- **Client Components** bersifat interaktif dan berjalan di klien. Anda memilih untuk menggunakannya dengan menambahkan direktif `'use client'` di bagian atas file Anda.

> **Peringatan:** Hati-hati saat mengambil data sensitif. Server Components adalah tempat yang bagus untuk ini, tetapi jangan pernah mengekspos kunci rahasia atau token ke Client Components.
