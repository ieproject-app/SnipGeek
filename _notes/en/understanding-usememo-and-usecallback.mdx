## Introduction

In React application development, performance is key. Two hooks that often come up when discussing optimization are `useMemo` and `useCallback`. Both help prevent unnecessary calculations or renders, but they are often misunderstood. Let's break them down.

## What is Memoization?

Before diving into the hooks, it's important to understand the core concept: **memoization**. This is an optimization technique where the results of expensive function calls are cached. If the function is called again with the same inputs, the cached result is returned without re-executing the function.

`useMemo` and `useCallback` are React's implementations of this concept.

## `useMemo`: Remembering a Value

Use `useMemo` when you have an expensive calculation that you don't want to run on every render.

**When to use it?**
- When computing a value from a large array (filtering, sorting).
- When performing complex calculations whose results don't change unless their dependencies do.

**Example:**

```jsx
import React, { useMemo, useState } from 'react';

function TodoList({ todos, filter }) {
  const [newTodo, setNewTodo] = useState('');

  // This calculation will only run again if 'todos' or 'filter' changes.
  const visibleTodos = useMemo(() => {
    console.log("Running expensive calculation...");
    return todos.filter(todo => todo.text.includes(filter));
  }, [todos, filter]);

  return (
    <div>
      {/* ... */}
      <ul>
        {visibleTodos.map(todo => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
    </div>
  );
}
```

Without `useMemo`, `todos.filter(...)` would run every time the `TodoList` component re-renders, even if it's caused by an irrelevant state change like `newTodo`.

## `useCallback`: Remembering a Function

Use `useCallback` to memoize a function definition so it isn't recreated on every render.

**When to use it?**
- When you are passing a callback function to an optimized child component (e.g., one wrapped in `React.memo`).
- Without `useCallback`, the callback function would be a new object on every render, causing the child component to re-render even if its other props haven't changed.

**Example:**

```jsx
import React, { useCallback, useState } from 'react';

const Button = React.memo(({ onClick }) => {
  console.log("Button re-rendered");
  return <button onClick={onClick}>Click Me</button>;
});

function ParentComponent() {
  const [count, setCount] = useState(0);

  // This function will only be recreated if its dependencies (none in this case) change.
  const handleClick = useCallback(() => {
    // do something
    console.log("Button clicked!");
  }, []);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
      <Button onClick={handleClick} />
    </div>
  );
}
```

In this example, when you click the "Increment" button, `ParentComponent` re-renders. Because `handleClick` is wrapped in `useCallback`, its function reference remains the same. As a result, the `Button` component does not re-render unnecessarily.

## Conclusion

- **`useMemo`** returns a memoized **value**.
- **`useCallback`** returns a memoized **function**.

Both are powerful tools, but don't overuse them. Use them only when you have identified a performance bottleneck. Premature optimization can make code more complex without providing any real benefit. Always measure your app's performance before and after applying these optimizations.
