## Pengantar

Dalam pengembangan aplikasi React, performa adalah kunci. Dua *hooks* yang sering muncul saat membahas optimisasi adalah `useMemo` dan `useCallback`. Keduanya membantu mencegah kalkulasi atau render yang tidak perlu, tetapi seringkali disalahpahami. Mari kita bedah keduanya.

## Apa itu Memoization?

Sebelum masuk ke *hooks*, penting untuk memahami konsep dasarnya: **memoization**. Ini adalah teknik optimisasi di mana hasil dari pemanggilan fungsi yang mahal (memakan waktu) disimpan dalam *cache*. Jika fungsi tersebut dipanggil lagi dengan input yang sama, hasil yang tersimpan akan dikembalikan tanpa perlu mengeksekusi ulang fungsi tersebut.

`useMemo` dan `useCallback` adalah implementasi dari konsep ini di dalam React.

## `useMemo`: Mengingat Nilai

Gunakan `useMemo` ketika Anda memiliki kalkulasi yang mahal yang tidak ingin Anda jalankan di setiap render.

**Kapan menggunakannya?**
- Saat menghitung nilai dari sebuah array besar (memfilter, menyortir).
- Saat melakukan kalkulasi kompleks yang hasilnya tidak berubah kecuali dependensinya berubah.

**Contoh:**

```jsx
import React, { useMemo, useState } from 'react';

function TodoList({ todos, filter }) {
  const [newTodo, setNewTodo] = useState('');

  // Kalkulasi ini hanya akan berjalan kembali jika 'todos' atau 'filter' berubah.
  const visibleTodos = useMemo(() => {
    console.log("Menjalankan kalkulasi yang mahal...");
    return todos.filter(todo => todo.text.includes(filter));
  }, [todos, filter]);

  return (
    <div>
      {/* ... */}
      <ul>
        {visibleTodos.map(todo => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
    </div>
  );
}
```

Tanpa `useMemo`, `todos.filter(...)` akan berjalan setiap kali komponen `TodoList` di-render ulang, bahkan jika itu disebabkan oleh perubahan *state* `newTodo` yang tidak relevan.

## `useCallback`: Mengingat Fungsi

Gunakan `useCallback` untuk mengingat definisi sebuah fungsi, sehingga fungsi tersebut tidak dibuat ulang di setiap render.

**Kapan menggunakannya?**
- Saat Anda memberikan fungsi *callback* ke komponen anak yang dioptimalkan (misalnya, dibungkus dengan `React.memo`).
- Jika tidak menggunakan `useCallback`, fungsi *callback* akan menjadi objek baru di setiap render, menyebabkan komponen anak ikut di-render ulang meskipun *props* lainnya tidak berubah.

**Contoh:**

```jsx
import React, { useCallback, useState } from 'react';

const Button = React.memo(({ onClick }) => {
  console.log("Tombol di-render ulang");
  return <button onClick={onClick}>Klik Saya</button>;
});

function ParentComponent() {
  const [count, setCount] = useState(0);

  // Fungsi ini hanya akan dibuat ulang jika dependensinya (dalam hal ini tidak ada) berubah.
  const handleClick = useCallback(() => {
    // lakukan sesuatu
    console.log("Tombol diklik!");
  }, []);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
      <Button onClick={handleClick} />
    </div>
  );
}
```

Dalam contoh ini, saat Anda menekan tombol "Increment", `ParentComponent` akan di-render ulang. Karena `handleClick` dibungkus dengan `useCallback`, referensi fungsinya tetap sama. Akibatnya, `Button` tidak akan di-render ulang secara tidak perlu.

## Kesimpulan

- **`useMemo`** mengembalikan **nilai** yang sudah di-memoize.
- **`useCallback`** mengembalikan **fungsi** yang sudah di-memoize.

Keduanya adalah alat yang ampuh, tetapi jangan digunakan secara berlebihan. Gunakan hanya ketika Anda telah mengidentifikasi adanya masalah performa. Optimisasi yang prematur bisa membuat kode lebih kompleks tanpa memberikan manfaat yang nyata. Selalu ukur performa aplikasi Anda sebelum dan sesudah menerapkan optimisasi ini.
