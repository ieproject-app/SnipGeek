Saat membangun blog ini, kami menemukan sebuah masalah visual yang cukup mengganggu: semua elemen `<code>` memiliki tampilan yang sama. Ini berarti potongan kode singkat di dalam paragraf (`inline code`) terlihat persis seperti blok kode multi-baris yang seharusnya memiliki latar belakang dan padding tersendiri.

### Akar Masalah

Secara default, kami memberikan gaya dasar pada tag `<code>` di file `globals.css`:

```css
code {
  @apply font-code relative rounded bg-muted px-[0.4rem] py-[0.2rem] font-mono text-sm font-semibold;
}
```

Masalahnya, pustaka *syntax highlighting* (Shiki) yang kami gunakan juga menghasilkan struktur HTML `<pre><code>...</code></pre>` untuk blok kode. Akibatnya, gaya `code` di atas juga diterapkan pada blok kode besar, menimpa gaya indah dari Shiki.

### Solusi: Menggunakan `@layer` dan Reset CSS

Solusinya adalah dengan membedakan penargetan CSS. Kami tetap mempertahankan gaya dasar untuk `code`, tetapi menambahkan sebuah *override* (penimpaan) khusus untuk `code` yang berada di dalam tag `<pre>`.

Kami menggunakan `@layer components` dari Tailwind CSS untuk mendefinisikan gaya khusus ini, dan menambahkan aturan reset:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer components {
  /* Gaya default untuk kode inline */
  code {
    @apply font-code relative rounded bg-muted px-[0.4rem] py-[0.2rem] font-mono text-sm font-semibold text-foreground/80;
  }

  /* Reset gaya khusus untuk blok kode dari Shiki */
  pre code {
    @apply bg-transparent text-inherit p-0 font-normal rounded-none static;
  }
}
```

Dengan aturan `pre code`, kami secara efektif "menghapus" gaya kustom dari `code` inline setiap kali ia berada di dalam sebuah blok `<pre>`, memungkinkan gaya dari Shiki untuk mengambil alih sepenuhnya. Hasilnya, kode inline dan blok kode kini memiliki tampilan yang berbeda dan sesuai fungsinya.
